# 定价口径与市场对比调查（t.wiselink.cc 网关 vs oken.ai 市场数据）

> 生成：2026-06-03。
> 数据来源：本仓库 `backend/data.db` + `backend/services/relay.py` 实测；oken.ai 比价 API 实拉；本机对 `t.wiselink.cc` 网关的真实延迟实测。
> **本文档只收录已验证为正确的数据。** 调查过程中一度出现的"官方9折"结论是错误的（把 T粒 数值误当美元），已剔除，不予记录。

## 1. 计费单位定性（决定性，已对账验证）

| 项 | 结论 | 证据 |
|---|---|---|
| 内部计价单位 | **T粒** | 前端 `StudentDashboard.tsx` 显示「T粒 余额」；`Models.tsx` 价格字段标「T粒/千T粒」 |
| 兑换汇率 | **1 T粒 = 0.01 元 = 1 分**（10元 = 1000 T粒） | `QuickGuide.tsx`「10元 = 1000 T粒」；`top_ups` 表 `amount=1000, payment_amount=10.0` |
| 价格字段单位 | **T粒 / 千 tokens** | `QuickGuide.tsx`「价格为 T粒/千 tokens」 |
| 计费公式 | `cost = (prompt/1000)*input_price + (completion/1000)*output_price` | `relay.py:56-57` |

**对账验证：** Sonnet 4.6 一次 8 prompt + 5 completion 的调用：
`(8/1000)×2.808 + (5/1000)×14.04 = 0.092664 T粒` —— 与 `request_logs.cost = 0.092664` **完全吻合**。

### 由此确认的两点（此前的疑虑均已排除）

- **`/1000` 不是 bug。** 价格单位本就是「T粒/千 tokens」，公式与 DB 扣费精确一致，无 1000× 多扣问题。
- **DeepSeek 价格单位正常。** 与 Claude 同口径（T粒/千 tokens），换算后落在合理区间（见 §4）。

### 单位换算公式

```
元 / 1M tokens = (price 数值，单位 T粒/千tokens) × 10
```

## 4. oken.ai 是什么（重要：不是网关）

- **性质**：AI 模型「价格 + 线路延迟」的**比价/监控聚合平台**，不卖 token、不提供可接入的 API。官方自述：*"a comparison and monitoring platform … we aggregate API pricing and latency data from official providers, cloud vendors, and third-party relay platforms"*。
- 与本项目的关系:本网关这类中转站，正是它会**收录排行的对象**。两者不是同类，无法做"网关 vs 网关"对比；正确做法是用它的市场数据给本网关**定位**。
- **比价 API**（实测可直连，无需鉴权）：
  - base：`https://gateway.oken.ai/v4/services/jiazhi-aipolymerization/api/v1`
  - `GET /models?page_size=500` → 全量 SKU + `min_price_info` / `official_price_info`
  - `GET /suppliers?sku_id={id}&pricing_method=1&page_size=200` → 某模型下各中转站明细
- **口径**：价格为 **人民币 / 1M tokens**，官方价 = `USD × 6.7626`（字段 `input_price_display` 直接写 `"$5 x 6.76"`）；`response_time` 为**毫秒**延迟（oken 探针节点测得，仅供参考）。

## 6. 本网关 vs 市场（正确口径对比，元/1M tokens）

| 模型 | 本网关 输入/输出 | 官方直连 输入/输出 | 对官方倍数 | 市场中位 输入/输出 |
|---|---|---|---|---|
| Opus | 46.8 / 234 | 33.81 / 169.07 | **×1.38** | 15.75 / 78.75 |
| Sonnet | 28.08 / 140.4 | 20.29 / 101.44 | **×1.38** | 7.80 / 39.0 |
| Haiku | 9.36 / 46.8 | 6.76 / 33.81 | **×1.38** | 1.43 / 7.15 |
| DeepSeek-V4-Pro | 2.6 / 10.4 | — | — | 3.0 / 6.0 |
| deepseek-v4-flash | 1.3 / 2.6 | — | — | 1.0 / 2.0 |

倍率来源：`0.936(种价系数) × 10(元/千→元/1M，因 T粒=0.01元) ÷ 6.7626(oken 汇率口径) ≈ 1.38`，Claude 三档一致。

### 定位结论

- **价格**：Claude 系实际收费 = **Anthropic 官方直连 × 1.38（约 38% 加价）**，落在 oken 全市场约 **90 分位**（最贵的前 ~10%），高于官方直连、约为市场中位价的 3 倍。
  - **此 38% 加价为有意设置的经销毛利，当前不调整**（2026-06-03 决策）。计价与计费逻辑本身正确，无需修改。
- **DeepSeek**：换算后落在市场中位附近（flash 略高于中位、pro 输入低于/输出高于中位），定价合理。
- **速度**：本网关 ~1.7s，**优于市场中位**（Opus 市场中位 6.6s），与官方直连(2.0s)相当或略快，处于第一梯队。低价号普遍以稳定性/速度换价格。
- **综合象限**：「准官方直连」——价格在官方之上(×1.38)、速度/稳定性第一梯队。卖的是稳定与省心，不与灰产地板价竞争；与"学生/教学场景稳定官方直连"的项目定位自洽。
